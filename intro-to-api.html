Intro to the USDA API

View the USDA APIs website here

What is an API?

An API, or Application Program Interface, is a set of methods that makes it easy to communicate information from one location to another. APIs serve many different purposes. Some examples are querying a remote database, calling a method from a software library, or providing an interface between to your operating system.

Often times, an API will require a key for access. The key serves as an identifier so that the provider can track how their API is being. It is common for an API key to have a limit on how many requests that key can make in a given time period.

How to get a USDA API key

Go to their API homepage, https://ndb.nal.usda.gov/ndb/api/doc, and scroll down to Gaining Access and click Sign up now. Then fill out the API Signup form and click submit and you'll receive your API key.

What can you use the USDA APIs for?

The USDA APIs provide access to their Food Composition Database. This database gives you access to over 180,000 food items from AARDVARK HABENERO HOT SAUCE to ZYMBOM, ENERGY DRINK WITH TAURINE and all the nutritional information, serving sizes, and ingredients.

USDA APIs

  • Food Reports: A Food Report is a list of nutrients and their values in various portions for a specific food
  • List: List requests provide a paged browse and select experience
  • Nutrient Reports: A Nutrient Report is a list of foods and their nutrient values for a set of specified nutrients
  • Search: Search sends keyword queries and returns lists of foods which contain one or more of the keywords

This guide will target the Search and Food Report APIs.
