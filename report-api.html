What is in the Food Report?

The food report contains a lot of useful information about the food that you submit to it. This information is so valuable that you might say it's the "meat and potatoes" of the USDA APIs.

Some of the most important things in the report are:
  • Ingredients
  • Serving Size
  • Nutrients
    • For 100 grams (whether or not this is a serving size you'll find)
    • For the default serving size
    • For other serving sizes (if available)

Using the Food Report API

In this section, we are going to go through the process of setting up a call to the Food Report API. Because a lot of this process is the same as setting up a call to the Search API, we are only going to point out what's different.

            var url = "https://api.nal.usda.gov/ndb/reports/V2?api_key=";

var payload = {
  "ndbno": "01009",
  "type": "f"
};

That is all that is different in this API call. First, the url where you send the call is different. The payload is slightly different. It includes an "ndbno" that indentifies the food you want the report on (you get "ndbno" from the Seach API call). In the payload, you also say what kind of report you want. There is "b" (Basic Report), which contains a limited set of common nutrients. There is "f" (Full Report), which contains all the nutrients available for that food. Lastly, there is "s" (Statistics Report), which provides a lot of statistical parameters from studies that include parameters like upper and lower error bounds data from studies on that food.
